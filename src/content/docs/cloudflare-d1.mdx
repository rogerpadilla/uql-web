## Overview

UQL supports **Cloudflare D1** so you can run a fully type-safe ORM on **Cloudflare Workers** with SQLite under the hood.

This guide covers:

- Wiring UQL to a D1 database.
- Querying from a Worker handler.

---

## 1. Bind a D1 database

In your Cloudflare configuration (for example `wrangler.toml`):

```toml
[[d1_databases]]
binding = "DB"
database_name = "uql-app"
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

Your Worker will receive a `DB` binding at runtime.

---

## 2. Define entities

```ts
import { Entity, Id, Field } from 'uql-orm';

@Entity()
export class Todo {
  @Id()
  id?: number;

  @Field()
  title?: string;

  @Field({ type: 'boolean', defaultValue: false })
  completed?: boolean;
}
```

---

## 3. Query D1 from the Worker

The D1 binding arrives per request via `env`, so create the pool from it inside the handler (it is a cheap wrapper; `end()` is a no-op for D1 bindings). Importing your entities module registers the decorated entities:

```ts
import { type D1Database, D1QuerierPool } from 'uql-orm/d1';
import { Todo } from './entities';

export default {
  async fetch(request: Request, env: { DB: D1Database }) {
    const pool = new D1QuerierPool(env.DB);

    const todos = await pool.findMany(Todo, {
      $select: { id: true, title: true, completed: true },
      $limit: 50,
    });

    return Response.json(todos);
  },
};
```

This pattern works with any D1-backed data model. To expose entities as REST endpoints from the same Worker, mount [`createFetchHandler`](/extensions-http) next to your routes; for query capabilities, see the [Querying](/querying/querier) docs.
